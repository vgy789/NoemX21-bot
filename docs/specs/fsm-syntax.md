# Спецификация синтаксиса FSM

Данный документ описывает структуру YAML-файлов, используемых для определения логики конечного автомата (Finite State Machine) бота.

## Общая структура (FlowSpec)

Каждый файл потока определяет набор состояний и переходов.

| Поле | Тип | Обязательное | Описание |
| :--- | :--- | :--- | :--- |
| `meta` | `Meta` | Да | Метаданные потока. |
| `initial_state` | `string` | Да | Идентификатор начального состояния. |
| `states` | `map[string]State` | Да | Карта состояний потока. |

### Meta

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `feature_id` | `string` | Уникальный ID функциональности (например, `REG-001`). |
| `name` | `string` | Человекочитаемое название. |
| `description` | `string` | Краткое описание назначения потока. |
| `version` | `string` | Версия схемы (например, `2.2.0`). |

---

## Состояние (State)

Состояние — это минимальный блок логики, который либо взаимодействует с пользователем, либо выполняет системную операцию.

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `type` | `string` | Тип состояния: `interactive`, `system`, `input`, `final`. |
| `description` | `string` | Описание роли состояния (для документации). |
| `interface` | `Interface` | Настройки визуального отображения (текст, кнопки). |
| `logic` | `Logic` | Системное действие, выполняемое в состоянии. |
| `validation` | `Validation` | Правила проверки ввода (для типа `input`). |
| `transitions` | `[]Transition` | Список возможных переходов в другие состояния. |

### Типы состояний

1.  **`interactive`**: Отображает сообщение и кнопки. Ожидает нажатия кнопки пользователем.
2.  **`system`**: Выполняет `logic.action` и автоматически переходит по условию (`condition`). Не отображается пользователю напрямую (сквозное).
3.  **`input`**: Ожидает текстовый ввод от пользователя. Проверяет его через `validation`.
4.  **`final`**: Конечное состояние потока. Не предполагает дальнейших переходов без инициации нового процесса.

## Интерфейс (Interface)

Определяет, что видит пользователь в Telegram.

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `text` | `map[string]string` | Локализованный текст сообщения. Ключи: `ru`, `en`. |
| `error_invalid` | `map[string]string`| Текст ошибки при невалидном вводе (для `input`). |
| `buttons` | `[]Button` | Список кнопок под сообщением. |

### Кнопка (Button)

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | `string` | Технический ID (отправляется как `callback_data`). |
| `label` | `string` \| `map` | Текст на кнопке (строка или карта локализации). |
| `next_state` | `string` | Состояние, в которое нужно перейти при нажатии. |
| `row` | `int` | (Опционально) Номер строки для группировки кнопок. |

---

## Логика и Действия (Logic)

Используется преимущественно в состояниях типа `system`.

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `action` | `string` | Имя зарегистрированного в Go-коде действия. |
| `payload` | `map` | Параметры, передаваемые в действие. Поддерживают подстановку переменных `$context.key`. |

---

## Переходы (Transition)

Определяют правила навигации.

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `next_state` | `string` | Целевое состояние. |
| `condition` | `string` | Условие (для `system` состояний), например: `registered == true`. |
| `trigger` | `string` | Событие (для `input` состояний), например: `on_valid_input`. |

---

## Переменные и Шаблоны

Движок FSM поддерживает динамическую подстановку данных в текст сообщений, подписи кнопок и параметры действий (`payload`).

### Синтаксис подстановки

| Формат | Описание | Пример |
| :--- | :--- | :--- |
| `{key}` | Основной формат для текста и кнопок. | `Привет, {name}!` |
| `$context.key` | Формат для использования в `logic.payload`. | `login: $context.last_input` |
| `$updates.key` | Алиас для `$context.key`. | `$updates.status` |

### Стандартные переменные

| Переменная | Описание |
| :--- | :--- |
| `{last_input}` | Последний текст, введенный пользователем (для `input` состояний). |
| `{attempts}` | (Если реализовано в экшене) Код попыток ввода. |

## Примеры

### Интерактивный выбор
```yaml
SELECT_LANGUAGE:
  type: interactive
  interface:
    text:
      ru: Выберите язык:
      en: Choose language:
    buttons:
      - id: set_ru
        label: 🇷🇺 Русский
        next_state: START
      - id: set_en
        label: 🇺🇸 English
        next_state: START
```

### Системная проверка
```yaml
START:
  type: system
  logic:
    action: is_user_registered
  transitions:
    - condition: registered == true
      next_state: MAIN_MENU
    - condition: registered == false
      next_state: INPUT_LOGIN
```

### Ввод данных
```yaml
INPUT_LOGIN:
  type: input
  validation:
    regex: ^[a-z]{4,9}$
  interface:
    text:
      ru: Введите логин:
    error_invalid:
      ru: Неверный формат логина!
  transitions:
    - trigger: on_valid_input
      next_state: VALIDATE_API
```
